## What Went Well

- **Reuse over reinvention.** No story invented a new pattern. 14.1 mirrored Sarah's gate (13.1); 14.4 reused the storageState SSO design (13.4); a single `agent_run_id` serves as the run id (no separate run table). Six stories landed with zero rework between them.
- **Disciplined artifact architecture.** Every output routes through the artifact service — no ad-hoc filesystem writes. The "visible doc + hidden companion" split (`report.md` `kind=report` + `report.json` `kind=configuration`) keeps the sidebar clean with no migration.
- **Security-first execution service.** Subprocess `pytest` isolation (`-o addopts= -p no:cov -p no:base_url`, fresh rootdir) sidesteps the coverage-gate and `pytest-base-url` ScopeMismatch traps. `scrub-at-source` + leak-canary tests across env-build, JUnit parse, scrub, messages, reports, and API responses — no path leaks the captured `storageState` blob or any secret.
- **Graceful degradation as a rule.** Every attachment/provenance lookup degrades to `null`/"(unavailable)"; a missing screenshot or unavailable browser never breaks the report. Hard-block when no captured session exists for `(env, role)` (Decision #3, confirmed) — execution never silently runs unauthenticated.

## Challenges & Growth Areas

- **Hermetic-green ≠ integration-validated.** All backend tests mock `subprocess.run` with canned JUnit XML. The real-browser execution path — the most powerful capability in the epic — has **never run live** (needs reachable app + installed browsers + captured session). This repeats the Epic 12-13 lesson that green unit tests over mocks are insufficient for interactive/execution workflows. **Top open risk.**
- **Carry-over from 12-13 action items.** Item #4 (mandatory E2E UI coverage) is only partially met — E2E was regrouped into 7 flow-groups and live-validated 2026-06-21, but the Jack/step-5/14.6 surfaces have no live E2E. Item #5 (centralize frontend state) shows no evidence of being addressed within Epic 14 scope.
- **Release hygiene pending.** Whole changeset uncommitted; `alembic upgrade head` not run. Code review is done, but schema is not on the DB.

## Key Insights

1. **Track live validation explicitly for integration-heavy capabilities.** A hermetic suite proves logic, not behavior. When a capability can only be exercised end-to-end, the live run is a first-class deliverable — schedule it, don't assume it.
2. **Reuse-over-reinvent compounds.** Anchoring each story to an existing pattern (gate, storageState, artifact split) eliminated rework and kept the review surface small.
3. **Scrub-at-source + leak-canary is the right foundation** for any subprocess that handles credentials — scrub once where data is produced, render as-is downstream.
4. **A prior retro's "next epic preview" can go stale after a reprioritization.** The 12-13 retro predicted Epic 14 = Audit/Metrics; the 2026-06-20 roadmap reorder made it Test Execution. Re-derive the next-epic context from `epics.md`, not the previous retro.
