## Story

As a maintainer of the air-gapped UAT environment,
I want a working `model_transfer` tool to export model/benchmark data from an internet-connected machine and apply it on UAT,
so that the benchmark scores can be refreshed offline without relying on the air-gapped host reaching llm-stats.

### Observed gap

Migration [`273b69541e94`:11](alembic/versions/273b69541e94_seed_model_benchmarks_refresh.py:11) docstring says it was *"Auto-generated by `python -m ai_qa.admin.model_transfer make-migration`"*, but `src/ai_qa/admin/model_transfer.py` is **missing from source and git history** — only a gitignored `.pyc` remains ([src/ai_qa/admin/](src/ai_qa/admin/) has just `__init__.py` + `model_sync.py`). The CLI would fail with `ModuleNotFoundError`. See [[uat-airgapped-egress-model-transfer]].

## Acceptance Criteria

1. **Tool exists + documented.** Given the repo, when a maintainer runs `python -m ai_qa.admin.model_transfer --help`, then the command exists with documented subcommands (at least `make-migration` matching what produced `273`).
2. **Faithful export.** Given an internet-connected machine with synced `discovered_models` + `model_benchmark_scores`, when `make-migration` runs, then it emits a refresh migration that is a TRUE replace-snapshot (deletes the full prior on-prem id set before insert — the bug 16-14 had to patch — so future refreshes do not orphan rows).
3. **Offline apply.** Given the generated migration (or an import path), when applied on the air-gapped UAT DB, then the scores match the source machine.
4. **Decision: remove vs restore.** If the team prefers not to maintain this tool, then the alternative is to delete the `273` docstring reference and document the seed-migration-only path; bring this choice to Thuong.
