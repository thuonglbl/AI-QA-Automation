## What Went Well

- **Execution Report Harmonization:** Epic 16 successfully folded the ad-hoc Jack Execution Report components built during Epic 14 into the centralized Conversational Shell (Story 16.4). 
- **Storage Flattening without Regression:** The removal of per-role sub-folders (16.10) for test scripts was executed smoothly. Because Jack already read the role from the JSON sidecar, the transition to flat storage required only whole-folder disambiguation, preventing any execution regressions.
- **Rapid Stabilization:** When severe integration bugs emerged (Sarah auth failure, DB pool leaks, event loop blocks), the team quickly pivoted to create and execute targeted stories (16.12-16.18) to stabilize the core platform before release.

## Challenges & Growth Areas

- **Agent Initialization Lifecycle Constraints:** The `16.12` Sarah auth bug revealed a major blind spot: caching `self.config` at agent instantiation time captures an empty API key, as the user's project context (and secrets) are attached later in the lifecycle. This necessitated porting the idempotent `_ensure_llm_ready()` pattern across our agents.
- **Async/Sync Boundary Friction:** As WebSocket traffic and concurrent LLM streams increased, our synchronous DB calls and legacy synchronous LLM clients began starving the ASGI event loop (16.16). We had to carefully manage these boundaries to keep the application responsive.
- **Live Validation Bottleneck:** We are still accumulating features (Admin Dashboard from Epic 15, Jack E2E execution from Epic 14, and now the entire Conversational Shell) that are hermetic-green but have not been validated against the live Win11 + PostgreSQL target environment.

## Key Insights

1. **Never capture secret-dependent configuration at agent construction:** The project context is injected post-instantiation. Configuration that depends on user secrets must be resolved right before execution (e.g., via `_ensure_llm_ready()`).
2. **Hermetic testing masks lifecycle bugs:** Our unit tests mocked the LLM config too early, bypassing the real instantiation flow and hiding the Sarah auth bug from the CI pipeline. 
3. **ASGI Event Loop demands strict sync isolation:** In a highly concurrent WebSocket environment, synchronous LLM clients or blocking DB queries will freeze the shell. These must be wrapped in `run_in_threadpool` or migrated to pure async.
